⚡ Deployment Options

1️⃣ Vercel (Recommended)

Deploy in minutes with zero configuration:

  1. Push code to GitHub
  2. Go to vercel.com
  3. Click "New Project" → Import GitHub repository
  4. Vercel auto-detects vercel.json configuration
  5. Click "Deploy" and your app goes live
✅ Vercel Benefits: Free tier, auto-scaling, HTTPS, custom domain, analytics dashboard

2️⃣ Docker

Containerized deployment for any platform:

# Build image
docker build -t star-office-ui .

# Run locally
docker run -p 18791:18791 star-office-ui

# Or use Docker Compose
docker-compose up

Deploy to: Railway, Render, Fly.io, AWS ECS, DigitalOcean, or any Docker-compatible platform

3️⃣ Traditional Server (VPS)

Deploy to Linux VPS with Nginx reverse proxy:

# 1. Install dependencies
sudo apt-get install python3 python3-pip nginx

# 2. Install Python packages
pip3 install -r backend/requirements.txt

# 3. Configure Nginx (proxy to port 18791)
# 4. Start Flask app
python3 backend/app.py

🌐 Available Routes

🔧 Local Development

Setup

# Clone repository
git clone https://github.com/you/star-office-ui.git
cd star-office-ui

# Create virtual environment
python3 -m venv venv
source venv/bin/activate  # or: venv\Scripts\activate (Windows)

# Install dependencies
pip install -r backend/requirements.txt

# Run server
python backend/app.py

Access at:

  • http://localhost:18791/landing (landing page)
  • http://localhost:18791/ (dashboard)
  • http://localhost:18791/join (join form)

🎮 Using the Dashboard

Main Features

  • Status Buttons: Change Star's state (Standby, Writing, Researching, etc.)
  • Guest List: Monitor agents in real-time
  • Memo Panel: View yesterday's journal entry
  • Join Key: Agents use this to connect
  • Approval System: Control who joins

Agent States

State Location Meaning
Standby Breakroom Available, waiting for tasks
Writing Desk Working/organizing
Researching Desk Gathering information
Executing Desk Running processes
Syncing Server Room Backing up/synchronizing
Error Error Zone Something went wrong

🔐 Security

⚠️ Important: By default, the app uses local file storage. For production:
  • Use environment variables for sensitive data
  • Implement HTTPS (Vercel does this automatically)
  • Rotate join keys regularly
  • Use strong passwords for agent access
  • Consider database migration for state persistence

📊 API Reference

Get Agents

GET /agents
Response: [{ agentId, name, state, area, authStatus, ... }]

Join Agent

POST /join-agent
Body: { name, joinKey, state?, detail? }
Response: { ok, agentId, authStatus }

Push Status

POST /agent-push
Body: { agentId, joinKey, state, detail? }
Response: { ok, agentId, area }

🐛 Troubleshooting

Port already in use

# Change port in app.py:
app.run(host="0.0.0.0", port=8000)  # Change 18791 to 8000

Images not loading

Ensure frontend/ directory is properly served via /static/ routes.

Agents not appearing

Check if agents-state.json exists and has valid permissions.
